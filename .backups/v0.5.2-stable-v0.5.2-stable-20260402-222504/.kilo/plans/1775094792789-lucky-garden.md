# DIAGNOSIS PLAN: Find & Remove the "0" After Title

## PROBLEM STATEMENT

**Symptom**: The number "0" appears immediately after the auction house name, on the same line, in compact view.

**Example**: "🏢 Direct Wholesale / Bulk Purchase 0"

**Location**: Between the name and the action buttons ("Set Default · Details")

**Affected Configs**: ALL configs with buyer_premium_pct = 0:
- Direct Wholesale / Bulk Purchase
- Private Seller / Kijiji  
- Facebook Marketplace
- Custom/Reseller Buy Cost

**NOT affected**: Encore Auctions (16%) and Estate Sale (10%)

## INVESTIGATION FINDINGS

### What We Know:

1. **API Response is Clean** - Backend returns separate fields:
   ```json
   {
     "name": "🏢 Direct Wholesale / Bulk Purchase",
     "buyer_premium_pct": 0.0
   }
   ```
   - Name is NOT concatenated with buyer_premium_pct

2. **Database Values Correct** - SQLite shows:
   ```sql
   id=2, name="🏢 Direct Wholesale / Bulk Purchase", buyer_premium_pct=0.0
   ```

3. **JSX Title Logic**:
   ```jsx
   <span className="text-sm font-medium text-surface-200">
     {h.name.replace(/\s*\d+%$/, '')}
   </span>
   ```
   - Regex only removes trailing "digits%"
   - Doesn't explain why "0" appears

4. **Conditional Rendering Logic**:
   ```jsx
   {h.buyer_premium_pct > 0 && (
     <span className="text-surface-400">
       <span className="text-surface-500">Premium:</span>
       <span className="text-brand-400"> {h.buyer_premium_pct}%</span>
     </span>
   )}
   ```
   - When buyer_premium_pct = 0, this doesn't render (CORRECT)
   - But somehow "0" still appears

5. **No Hidden Characters** - Byte-by-byte analysis shows no hidden content

## IMPLEMENTATION PLAN

### Approach A: Defensive Removal (SAFE)

**Strategy**: Hide any trailing number in the name display.

**Change**:
```javascript
// Current:
{h.name.replace(/\s*\d+%$/, '')}

// Change to:
{h.name.replace(/\s*\d+%?\s*\d*$/, '').trim()}
```

**Pros**: Removes any trailing numbers, safe
**Cons**: Doesn't address root cause, might hide legitimate numbers

### Approach B: Debug First, Fix Later (RECOMMENDED)

**Strategy**: Add debug logging to identify source.

**Change**:
```javascript
{console.log('Rendering:', h.id, h.name, h.buyer_premium_pct)}
{h.name.replace(/\s*\d+%$/, '')}
```

**Pros**: Identifies exact source
**Cons**: Requires user to check console

### Approach C: Conditional Wrapper (SAFEST)

**Strategy**: Wrap name in a component that filters out numbers.

**Change**:
```javascript
<span className="text-sm font-medium text-surface-200">
  {h.name.replace(/\s*\d+%$/, '').replace(/\s+\d+$/, '')}
</span>
```

**Pros**: Handles both "%X" and " X" cases
**Cons**: Regex complexity

### Approach D: Complete Audit (THOROUGH)

**Strategy**: Rewrite the entire title section from scratch with no ambiguity.

**Change**:
```javascript
<div className="flex items-center gap-2 mb-1">
  <span className="text-sm font-medium text-surface-200">
    {h.name.split(' ').filter(word => !word.match(/^\d+$/)).join(' ')}
  </span>
  {/* ... badges ... */}
</div>
```

**Pros**: Guaranteed to remove numbers
**Cons**: Might remove legitimate numbers in names

## RECOMMENDED FIX

**Best Approach**: Approach C (Conditional Wrapper)

**Reasoning**:
1. Safest - doesn't break existing code
2. Handles multiple edge cases
3. Simple regex change
4. No risk of breaking other features

**Implementation**:

**File**: `frontend/src/pages/Settings.jsx`
**Line**: 266

**Current Code**:
```javascript
<span className="text-sm font-medium text-surface-200">{h.name.replace(/\s*\d+%$/, '')}</span>
```

**New Code**:
```javascript
<span className="text-sm font-medium text-surface-200">{h.name.replace(/\s*\d+%$/, '').replace(/\s+\d+$/, '')}</span>
```

**Explanation**:
- First `.replace(/\s*\d+%$/, '')` removes trailing "digits%" (e.g., "16%")
- Second `.replace(/\s+\d+$/, '')` removes trailing "digits" (e.g., "0")
- This handles both cases without breaking anything
