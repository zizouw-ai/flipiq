# Deployment Plan: FlipIQ (Local to Online)

This plan outlines the steps to deploy the current local version of FlipIQ (FastAPI backend + React frontend) to Railway.

## Project Overview
- **Backend**: FastAPI (Python)
- **Frontend**: React (JavaScript/TypeScript)
- **Database**: SQLite (local) -> Target: Railway Volume at `/data/flipiq.db`
- **Platform**: Railway (Hobby Plan)
- **Domain**: `flipiq.ca`

## Deployment Strategy
We will use the Railway CLI and Git-integrated deployment.

### Phase 1: Infrastructure Configuration
1. **Railway Configuration**:
   - Review and update `railway.toml` for both backend and frontend services.
   - Verify `Procfile` for correct FastAPI startup command (uvicorn).
   - Configure `nixpacks.toml` if custom build steps are needed.

2. **Environment Variables**:
   - Identify all required `.env` variables for both frontend and backend.
   - Set these variables in the Railway dashboard or via `railway variables`.

3. **Persistence**:
   - Configure a Railway Volume mounted at `/data` for the backend service.
   - Ensure the backend is configured to store the SQLite database at `/data/flipiq.db`.

### Phase 2: Backend Deployment
1. **Health Check**: Implement/verify the `/health` endpoint in FastAPI.
2. **CORS Configuration**: Update CORS settings to allow requests from the Railway-assigned frontend domain and `flipiq.ca`.
3. **Deployment**: Push changes to trigger the auto-build or use `railway up`.

### Phase 3: Frontend Deployment
1. **API Endpoint Update**: Update the frontend API base URL to point to the production backend URL.
2. **Build Optimization**: Ensure the React build process is optimized for static hosting on Railway.
3. **Deployment**: Push changes to trigger the auto-build.

### Phase 4: Finalization & Testing
1. **Domain Mapping**: Connect `flipiq.ca` to the Railway services.
2. **Smoke Testing**: Verify end-to-end functionality (Login, Dashboard, Data Persistence).
3. **Logs Monitoring**: Use `railway logs` to debug any runtime issues.

## Verification Checklist
- [ ] Backend service is running and `/health` returns 200.
- [ ] Frontend is accessible via `flipiq.ca`.
- [ ] Database persists data across restarts (Volume check).
- [ ] CORS allows frontend-to-backend communication.
- [ ] All environment variables are correctly set in production.
