# Plan: Complete Frontend API Architecture Fix — Production Grade

## Problem Statement
Frontend cannot communicate with backend on Railway. Everything returns 405 errors. Root causes:

1. **`api.js` is STRIPPED** — truncated from 208 lines to 6 lines, all API function exports lost
2. **Double `/api` prefix** — hardcoded URLs include `/api`, then code appends another `/api`
3. **Conflicting config mechanisms** — three parallel approaches fighting:
   - `VITE_API_URL` (Vite build-time)
   - `BACKEND_URL` + nginx proxy_pass (runtime envsubst)
   - Hardcoded URLs in JS (`api.js`, `authStore.js`, `Profile.jsx`)
4. **railway.toml has empty values** — both `VITE_API_URL` and `BACKEND_URL` are `""`

## Architecture Decision: Single Source of Truth

**Decision**: Use a **single `config.js` module** (production-grade pattern).

All other files import from it. One place to change the backend URL.

```
frontend/src/config.js   ← Single source of truth for API URL
frontend/src/api.js      ← All API functions (restore + fix imports)
frontend/src/store/authStore.js ← Auth (imports config)
frontend/src/pages/Profile.jsx  ← Profile (imports config)
```

**How it works:**
- Vite build-time: `VITE_API_URL` is set in `railway.toml` build args
- Config module reads `import.meta.env.VITE_API_URL` at build time
- All API calls use the config module
- No hardcoded URLs anywhere else
- No nginx proxy passthrough for API (CORS handles it)
- Nginx serves static files only

## Implementation Steps

### Step 1: Create `frontend/src/config.js` (Single Source of Truth)

**File**: `frontend/src/config.js` (NEW)
```javascript
const isLocalhost = window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1'
const VITE_API_URL = import.meta.env.VITE_API_URL || ''

export const API_BASE_URL = isLocalhost
  ? ''                         // Dev: relative path, Vite proxy forwards
  : VITE_API_URL               // Production: baked in at build time

export const API_URL = API_BASE_URL.replace(/\/$/, '') + '/api'

// Helper function for all fetch calls
export async function apiFetch(path, options = {}) {
  const url = `${API_URL}${path}`
  const res = await fetch(url, {
    headers: { 'Content-Type': 'application/json', ...options.headers },
    ...options,
  })
  
  if (!res.ok) {
    const err = await res.json().catch(() => ({}))
    throw new Error(err.detail || err.message || `API Error ${res.status}`)
  }
  return res.json()
}
```

## Tradeoffs

| Approach | Pros | Cons |
|----------|------|------|
| Build-time VITE_API_URL (CHOSEN) | Fast at runtime, simple, no proxy | Must rebuild to change URL |
| Runtime fetch + env.js | Change URL without rebuild | Extra network call, more complex |
| Nginx proxy | Same-origin requests | Complex, fragile proxy config |
| Hardcoded (previous attempt) | Works now | Fragile, no staging support |

This plan uses **build-time VITE_API_URL** which is the standard Vite/React pattern and production-grade for single-environment deploys.
